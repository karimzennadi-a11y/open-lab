# Guide de contribution — Open Lab

## Première contribution

1. **Fork** ce repo (bouton en haut à droite sur GitHub)
2. Clone ton fork localement : `git clone https://github.com/TON-USERNAME/open-lab.git`
3. Crée une branche : `git checkout -b ma-contribution`
4. Fais tes modifications
5. Commit : `git commit -m "description courte"`
6. Push : `git push origin ma-contribution`
7. Ouvre une **Pull Request** vers `karimzennadi-a11y/open-lab`

## Ouvrir une Issue

Utilise les Issues pour :
- Poser une question technique
- Signaler une erreur dans un document
- Proposer une amélioration de conception
- Suggérer un nouveau besoin utilisateur

## Format des fichiers

| Type | Format accepté |
|------|---------------|
| Schémas / Plans | SVG, DXF, STEP, PDF |
| Calculs | .xlsx, .ods, .ipynb, .py |
| Documents | Markdown (.md), PDF |
| Fabrication | STL, gcode, DXF |

## Crédit

Chaque contributeur est ajouté dans `contributions/` avec :
- Nom / pseudo
- Compétence apportée
- Date et version
